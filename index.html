TEF Innova

Reto

Innova School busca ayudar a sus alumnos de 4to de media a decidir qué carrera que quieren estudiar. El reto consta en diseñar una herramienta que los guie en este camino.

Rol y Equipo

Este fue un reto individual en el tuve la oportunidad de participar en todas las etapas del proceso UX.

Proceso

*En la fase de investigación se desarrollaron encuestas online, entrevistas personales y búsquedas en la web para conocer a fondo a los usuarios.

*Las herramientas que use en la parte de síntesis fueron: afinity map para definir el journey más crítico, user persona, problem statements, WMH y What if.

*Para el proceso de ideación se realizó benchmark, además de un feature list para definir las principales funcionalidades y con este terminado se inició el proceso de priorización para llegar el MVP.

*Se realizó un paper prototype incial y luego un prototipo en alta fidelidad diseñado en illustrator.

*Para el proceso de testing tuve el apoyo de dos niñas las cuales validaron algunas funcionalidades y ayudaron a la mejora del flujo.

Lecciones aprendidas

Debo tener mucho más cuidado con los contrastes.

Ibank

Reto

Un banco busca una forma de llegar a los jóvenes milenials, para ello requieren desarrollar un banco 100% digital.

Rol y Equipo

Este proyecto se realizó en dúo donde pude intervenir en todas las etapas del diseño UX junto a mi compañera de Laboratoria.

Proceso

*En la etapa de descubrimiento se realizó un BCM el cual nos ayudó a comprender los objetivos de negocio; se hicieron entrevistas a usuarios y stakeholders, además del benchmark

*Para el proceso de síntesis se hizo 2 journey map (el primero para realizar una transacción y el otro para hacer un reclamo), adicional a ello están los problem statements, HMW y what if teniendo en cuenta para su realización el user persona.

*Para idear una solución usamos el feature list y luego una priorización para definir el MVP.

*En el proceso de prototipado se inició con 2 paper prototype (uno por integrante) con el cual se llegó a un paper prototype final y posteriormente al prototipo de alta fidelidad hecho en figma.

*El testing se realizó con el paper prototype final, en base a él se hicieron los cambios necesarios para llegar al prototipo en alta fidelidad.

Lecciones aprendidas

Al hacer una transacción de cualquier tipo, el usuario desea siempre tener una confirmación de que el proceso se realizó con éxito, lo cual no había contemplado a pesar del research que se realizó.

Viaggio

Reto

El programa San Borja en Bici busca crear una herramienta digital, que facilite la inscripción de los nuevos usuarios, así como el acceso a las bicis, las estaciones, y los tiempos de préstamos de las bicis.

Rol y Equipo

Nuestro equipo estuvo formado por 2 UX y 3 frontend, equipo con el cual se logró un prototipo de alta fidelidad y un producto funcional que lograba completar el proceso de inscripción.

Proceso

*Iniciamos la etapa de descubrimiento realizando entrevistas a usuarios y stakeholders, además del benchmark para conocer aplicaciones similares y sus servicios.

*Para la síntesis de la información definimos los journey maps del registro de un nuevo usuario y el proceso de renovar tiempo, los cuales eran los más complicados para los usuarios, definir el user persona, problema statements, HMW y what if hizo que el proceso de síntesis terminara y se pueda llegar al siguiente paso.

*En el proceso de ideación se priorizaron las funcionalidades descritas en el feature list.

Para el prototipado se usó figma como herramienta principal, con lo cual se logró el prototipo de alta fidelidad que se requería

Lecciones aprendidas

Algo nuevo para nosotras en UX era saber la forma en que las frontend trabajaban con React, pues este leguaje es algo diferente al ya conocido javascript.

Movie Without Conflict

Reto

El reto propuesto en la hackaton interna de Laboratoria proponía buscar y solucionar un problema relacionado a las películas haciendo uso del API OMdb.

Rol y Equipo

El equipo estuvo formado por 4 integrantes, y mi principal función fue definir el prototipo y maquetar la solución en base al mismo.

Proceso

*Se realizaron entrevistas, y algunas encuestas.

*Por el corto tiempo disponible cada una de las integrantes definió un prototipo en papel y con las ideas extraídas de ello se realizó un prototipo en papel común.

*El prototipo de alta fidelidad se realizó en illustrator.

*La maquetación se realizó con bootstrap y el lenguaje usado fue jquery.

Lecciones aprendidas

El uso de las API puede ser todo un reto, pero ayuda en gran cantidad a encontrar una solución más rápida de lo habitual.

Data Dashboard

Reto

Este reto nace de la necesidad de los traiding manager de Laboratoria de tener la información de cada promoción para tener un mejor control de todos los datos y encaminar mejor a las chicas.

Rol y Equipo

Este reto se trabajó en duos y la principal función de ambas era entender la data, aplicarla y crear gráficos a partir de ella.

Proceso

*El primer paso fue conocer la base de datos, como estaba formada y sobre todo pensar cómo podíamos recorrerla.

*Una vez entendido este mundo nuevo para nosotras, se buscaba la forma de cumplir con los requerimientos dictados por Laboratoria y analizar cómo se podría hacer para mostrarlo en pantalla.

*Por último y no menos importante, fue todo un reto hacer uso de google charts.

Lecciones aprendidas

Hasta ese momento no había tenido la oportunidad de conocer una base de datos, definitivamente fue muy importante saber cómo usarla para tener los resultados que se requieren.
